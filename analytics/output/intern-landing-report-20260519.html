03진단 — 어디서, 왜 빠지는가

1. Hero에서 43%가 즉시 이탈 P0

위치: 0–10% 스크롤 · 손실: 161 → 92명 (-69명)

원인. 현재 Hero 헤드라인 "첫 단추를 잘못 끼우면 10년 뒤 반드시 후회합니다"는 인턴 페르소나(승급 앞둔 인턴, 갓 승급한 초급 디자이너)에게 회피 동기를 자극합니다. 그러나 인턴이 즉시 떠올리는 자기 문제는 "10년 뒤"가 아니라 "다음 달 승급 시험" 또는 "고객 앞에서 떨리는 손"입니다. 광고에서 본 카피와 페이지 첫 화면 카피의 매칭 정합성이 떨어지면 0~10초 안에 닫힙니다.

2. 가격 노출 직후 40% 폭락 P0

위치: 85–90% 스크롤 · 손실: 71 → 43명 (-28명)

원인. 280만원이라는 숫자가 페이지 거의 끝에서 단독으로 노출됩니다. 분납·ROI·비교 앵커가 없어 가격 충격을 흡수하지 못합니다. 사실 USP·FAQ 문서에는 이미 "20% 선납 기수 확정 → 잔여는 개강 전까지" 분납 조건이 정리되어 있으나, 가격 카드에 노출되지 않고 있습니다.

3. 후기·FAQ 구간 평균 체류 0초 P1

위치: 65–85% 스크롤 · 80명 도달하지만 거의 읽지 않음

원인. 후기가 익명 처리(김○○, 이○○, 최○○) + "※ 1기 모집 중 — 페르소나 기반 후보 후기"라는 자체 디스클레이머가 노출되어 있습니다. 방문자는 한눈에 "검증된 후기가 아니다"라고 판단하고 스킵합니다. 광고 카피의 신뢰를 페이지가 스스로 깎고 있습니다.

4. 커리큘럼 후반 흥미 상실 P1

위치: 60–65% 스크롤 · 평균 체류 28초 → 5초로 추락

원인. Curriculum이 PHASE 1~5의 동일 텍스트 카드로 반복됩니다. 시각적 변화가 없어 후반부에서 "더 볼 게 없겠다"는 판단으로 빠른 스크롤로 전환됩니다. 도해도·결과물 이미지 1장씩만 추가해도 끝까지 읽힐 가능성이 높습니다.

5. 폼 마찰 — 폼 도달자의 47%가 이탈 P2

위치: 90–100% 스크롤 · 손실: 43 → 23명

원인. 현재 폼은 5개 필드(이름·연락처·E-mail·미용경력·과정명) + 별도 약관 체크박스를 요구합니다. 마지막 마찰 지점에서 결정 비용이 너무 높습니다. 이메일과 과정명은 회신 단계에서 받아도 충분합니다.

근거: E-mail·과정명은 회신 단계에서 수집 가능. 폼 도달 43명 중 23명만 끝까지 본 데이터는 폼 자체가 마지막 마찰임을 시사.

P2 — 여유 있을 때

07의사결정 필요 항목

다음 5건은 실행 전 결정이 필요합니다.

  1. 광고 헤드라인 카피 통일 현재 메타 광고 헤드라인과 페이지 Hero의 매칭 상태 확인 후, 한 쪽을 다른 쪽에 맞춰 통일.
  2. 후기 처리 방향 (A / B / C) (A) 클래식 60기 검증 후기 차용 / (B) 도해도·작품 갤러리 대체 / (C) 결정 보류.
  3. 클래식 졸업생 후기 인용 동의 박성훈·김지원 등 인턴 페이지 노출에 대한 본인 동의 재확인 (Option A 선택 시).
  4. 1기 개강일 확정 시 카운트다운 노출. 미확정 시 정원 시각화로 대체.
  5. 폼 필드 축소 합의 E-mail·과정명 제거에 대한 CRM/영업 측 동의 (회신 단계 수집으로 대체).
