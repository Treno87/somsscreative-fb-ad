Section 05
문제점 진단
Health Score 35/100 — 4개 영역 세부 진단 결과
🔴
[FAIL] 예산 낭비 — classic_헤어타겟 광고세트
CTR 5.02%로 클릭 반응은 양호하지만 CPL ₩30,254 (캠페인 평균의 3배 초과 = 3x킬룰). 광고비 소진 대비 전환 0건에 가까운 수준. 랜딩페이지 연결 오류 또는 오디언스 미스매치 의심. 즉시 일시정지 후 랜딩 URL·타겟 재검토 필요.
🔴
[FAIL] 소재 부족 — 광고세트당 1~2개
Meta 알고리즘이 최적 소재를 탐색하려면 광고세트당 최소 5개+ 소재가 필요합니다. 현재 모든 광고세트가 1~2개 소재만 운영 중이어서 알고리즘 탐색 및 학습이 사실상 불가능한 상태. 단일이미지(1P) 추가 3~4개, 영상/UGC 소재 확보 권장.
🔴
[FAIL] 학습 단계 미달 — 전 광고세트
Meta 광고 알고리즘 학습 완료를 위해서는 광고세트당 주 50건 이상의 전환이 필요합니다. 현재 5개 광고세트 전부 주 2~5건 수준으로 학습 미완료 상태("학습 중" 또는 "학습 제한"). 캠페인 예산 증액(최소 40,000원+) 또는 전환 목표를 상위 퍼널(랜딩 조회 등)로 변경 검토.
🟡
[WARNING] 비활성 캠페인 2개 방치
"망했어요" 포함 2개 캠페인이 비활성 상태로 계정에 잔류 중. 직접적인 비용 낭비는 없으나 계정 구조 복잡성 증가 → 성과 분석 혼선 유발. 아카이브 또는 완전 삭제로 계정 정리 권장.
🟡
[WARNING] 5P 슬라이드 포맷 CPL 74% 비효율
슬라이드(5P) 광고의 평균 CPL ₩17,405는 단일이미지(1P) ₩9,992 대비 74% 높습니다. 슬라이드가 브랜딩 인지에는 유효할 수 있으나 리드 전환 목적에서는 현행 소재 기준 비효율. 슬라이드 소재 전면 교체(스토리 포맷, 노하우 시리즈 등) 또는 일시 중단 권장.
🔵
[INFO] iOS 데이터 손실 — CAPI 미도입
iOS 14+ ATT 정책으로 인한 픽셀 전환 데이터 손실이 발생 중일 가능성이 높습니다. 실제 리드 수는 보고된 14건보다 많을 수 있으며, CPL은 과대 측정됐을 수 있습니다. Conversions API(CAPI) 도입으로 서버 측 전환 데이터 보완을 중기적으로 권장합니다.

Appendix
데이터 한계 및 주의사항
  • 단기 데이터 편향: 본 보고서는 4일(2026-03-21~24)간의 데이터를 기준으로 합니다. 통계적 유의성 확보를 위해서는 최소 7~14일 이상의 데이터 누적이 권장됩니다. A/B 테스트 결론은 예비적 판단으로 활용하시고, 추가 데이터로 검증하시기 바랍니다.
  • iOS 전환 손실: iOS 14+ ATT 정책으로 인해 Meta 픽셀 기반 전환 추적은 실제 리드보다 적게 집계될 수 있습니다. 실제 리드 수와 CPL은 보고 수치보다 양호할 수 있습니다. CAPI 도입 전까지는 Meta Business Manager 보고서와 실제 CRM(Google Sheets) 리드 수를 교차 검증하시기 바랍니다.
  • 어트리뷰션 윈도우: 본 보고서는 Meta 기본 어트리뷰션(클릭 후 7일, 조회 후 1일) 기준입니다. 실제 의사결정 주기가 다를 경우 어트리뷰션 설정 변경을 검토하세요.
  • 학습 단계 영향: 현재 모든 광고세트가 학습 단계 미완료 상태입니다. 알고리즘 학습이 완료되면 CPL이 추가 개선될 수 있으며, 현재 데이터는 최적화된 성과가 아닙니다.
  • 외부 변수 미포함: 경쟁사 광고 집행 현황, 시즌 효과, SNS 유기적 도달 등 외부 변수는 본 보고서 분석 범위에 포함되지 않습니다.
  • 보고서 유효기간: 본 보고서는 생성일(2026-03-24) 기준이며, 광고 설정 변경 후에는 재분석이 필요합니다.
