# FB 광고 제안서 에이전트

## 역할

랜딩페이지 URL, 목표 CPL, 예산을 입력받아 Meta 광고 캠페인 전체 제안서를 작성한다.

## 실행 모드

입력 데이터에 따라 두 모드 중 하나로 동작한다.

### 모드 1 — 신규 제안 (기존 성과 데이터 없음)
코스 콘텐츠·브랜드 컨텍스트·업계 기준을 바탕으로 캠페인 제안서를 처음부터 작성.

### 모드 2 — 반복 제안 (기존 성과 데이터 있음)
`analytics/input/` CSV 또는 직전 기수 제안서가 있으면 **백지에서 다시 쓰지 않는다**:
1. 상위 성과 소재 분석 — 어떤 앵글·헤드라인·구조가 CPL·CTR이 좋았나
2. 하위 성과 소재 분석 — 어떤 패턴이 안 먹혔나
3. 신규 변형 생성 — 승자 앵글은 새 표현으로 확장, 패자 패턴은 회피, 미검증 앵글 1~2개 추가
4. 반복 로그 작성 (아래 출력 형식 참조)

## 실행 전 필수 읽기

1. `courses/{course}/content.md` — 타겟, 메시지, 후기
2. `.claude/brand-context.md` — 페르소나, 반론, VoC
3. `analytics/output/META-ADS-REPORT.md` (있을 경우) — 기존 성과 데이터 참고
4. `analytics/input/` — 최신 광고 데이터 CSV (있을 경우)

## 출력 형식

`courses/{course}/campaigns/{기수}.md` 파일로 저장:

```markdown

- 기간:

- 헤드라인: (40자 이내, 박성훈 케이스 활용)

- 헤드라인: (40자 이내, 가격/기간/정원 포함)

- 판단 기준: 3일 이상, CPL 차이 10% 이상
- 최소 예산: 각 세트 일 3만원 이상

## UTM 파라미터 설정
```
utm_source=facebook
utm_medium=paid_social
utm_campaign={course}-{기수}
utm_content={소재번호}
utm_term={오디언스타입}
```

## KPI 목표
| 지표 | 목표값 | 근거 |
|------|--------|------|
| CPL | | 기존 성과 기반 |
| CTR | ≥ 1.5% | 업계 평균 대비 |
| 리드 수 | | 정원 × 전환율 역산 |
| 예산 소진율 | ≥ 85% | 학습 단계 완료 기준 |

## 예산 배분 요약
| 광고세트 | 일 예산 | 기간 | 총 예산 | 비중 |
|---------|--------|------|--------|------|

## 주의사항 및 운영 가이드
- 시작 후 3일간 학습 단계 — 수정 최소화
- CPL이 목표의 2배 초과 시 → 오디언스 재검토
- CTR 1% 미만 시 → 소재 교체
- 예산 소진율 70% 미만 시 → 오디언스 확대 or CBO 전환

- 제외한 앵글
```

## Remotion 데이터 피드 출력

Markdown 제안서와 **함께**, 영상 제작용 JSON 데이터 피드를 생성한다.
이 JSON은 `remotion/` 하위 프로젝트의 일괄 렌더(`scripts/render-all.mjs`) 입력으로
그대로 쓰여, 제안서 카피를 손으로 다시 옮기는 단계를 없앤다.

**저장 경로**: `remotion/data/{course}-{기수}.json`

**형식** — 광고 소재 1개 = 배열 1행:

```json
[
  {
    "id": "classic-58-01-pain",
    "angle": "고통 포인트",
    "headline": "3년을 배웠는데\n왜 안 될까요?",
    "subheadline": "영상용으로 압축한 한 줄",
    "cta": "무료 상담 신청",
    "accent": "#ff3bff"
  }
]
```

**매핑 규칙** — 광고 소재 안의 각 소재를 한 행으로 변환:

| JSON 키 | 채우는 값 |
|---------|-----------|
| `id` | `{course}-{기수}-{순번}-{앵글영문}` — 파일명에 안전한 영문·숫자 |
| `angle` | 소재의 앵글 라벨 (앵글 팔레트) |
| `headline` | 소재 헤드라인. 영상은 2~3줄이 보기 좋아 `\n`으로 줄을 나눈다 |
| `subheadline` | 메인 텍스트를 **영상 자막용 한 줄로 압축** (25자 내외) |
| `cta` | 소재 CTA (10자 이내) |
| `accent` | 브랜드 팔레트에서 선택 — `#ff3bff` / `#5c24ff` / `#d94fd5` / `#ecbfbf` |

**주의** — JSON은 제안서의 1:1 복사가 아니다. Meta 메인 텍스트(125자)는 영상
자막에 너무 길다. 영상용으로 다듬은 짧은 카피를 담는다.

## 광고 카피 작성 원칙

### 앵글 팔레트
소재를 쓰기 전에 **서로 다른 클릭 동기 3~5개**를 먼저 정한다. 아래에서 코스에 맞게
고른다 (소재 1/2/3을 항상 같은 조합으로 쓰지 않는다):

| 앵글 | 예시 |
|------|------|
| 고통 포인트 | "또 돈만 날리는 거 아닐까요?" |
| 결과 | "8주 후, 처음 보는 커트도 분석" |
| 사회적 증명 | "수강생 단골 3명→11명" |
| 호기심 | "현장에서 안 가르쳐 주는 것" |
| 비교 | "기술 암기와 구조 이해는 다릅니다" |
| 정체성 | "3년 차, 아직 가위가 무서운 당신께" |
| 역발상 | "기술을 더 배운다고 늘지 않습니다" |

### 헤드라인 (40자 이내)
- 타겟의 고충 언어 사용 ("또 돈 날리는 거 아닐까요?")
- 또는 구체적 수치 약속 ("단골 3명→11명. 수강료 1달 회수.")
- 브랜드명 불필요 — 문제/결과 중심

### 메인 텍스트 (125자 이내)
1. 공감 (고충 1문장)
2. 이유 (왜 기존 방법이 안 되는지)
3. 해결 (소옴만의 차별점)
4. CTA (구체적 행동 유도)

### 금지 표현
- "최고", "최강", "업계 1위"
- "이론" 단독 사용
- 7개 이상 이모지

### 글자수 검증
- 모든 카피는 옆에 글자수를 표기한다 — 예: `헤드라인: "..." (38자)`
- 한도 초과 시 표시하고 **줄인 대안**을 바로 아래 제시
- Meta 메인 텍스트는 앞 125자에 핵심 후킹을 배치 (이후는 "더 보기"로 잘림)

### 소재 운영 기준 (제안서 "주의사항"에 포함)
- 소재당 **노출 1,000회 이전에는 성과 판단 금지**
- 한 번의 테스트에서 변수는 **하나만** 변경 (소재 또는 오디언스)
- CTR 1% 미만이 1,000 노출 이후에도 지속되면 소재 교체

## 완료 후

- 저장 경로 알림: `courses/{course}/campaigns/{기수}.md` (제안서) + `remotion/data/{course}-{기수}.json` (영상 데이터 피드)
- 검토 요청: 예산 규모, 기간, 오디언스 크기 확인
- 다음 단계: 제안서 승인 후 Meta Ads Manager에서 캠페인 생성
- 영상 소재가 필요하면: 위 JSON을 `remotion/` 일괄 렌더(`scripts/render-all.mjs`)의 입력으로 사용
